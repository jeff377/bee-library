# Bee.NET 4.12.0 — 詳細版本說明

[English](4.12.0.md) ・ [繁體中文](4.12.0.zh-TW.md) ・ [← CHANGELOG](../../CHANGELOG.zh-TW.md)

本檔是 [CHANGELOG.zh-TW.md](../../CHANGELOG.zh-TW.md) 中 `[4.12.0]` 條目的詳細版；主 changelog 每項只列一行重點，本檔保留完整敘事——每項變更的「為什麼」、限制與設計脈絡。

> Bee.NET 仍處 pre-stable 演進階段。本版主軸為「Avalonia 控件家族走向行動端」：`Bee.UI.Avalonia` 在手機／窄視窗下響應式，`Bee.Definition` 型別相容 AOT reflection-only XmlSerializer。兩者合起來，正是讓同一套共用 Avalonia UI 不改一行就能以 **iOS** 與 **Android** single-view head 執行的基礎（已於 Northwind demo 端到端驗證）。本版**無破壞性變更**——呼叫端語法與磁碟上的 XML 格式皆不變。

### 新增

- **`FormView` 響應式佈局（`Bee.UI.Avalonia`）** — 窄視窗下整張表單自適應：主檔欄位區由 `FormLayout.ColumnCount` 多欄重排為單欄，明細 `GridControl` 由 inline `InCell` 編輯改用「唯讀 grid + `EditForm`」流程。以單一 `_isCompact` 狀態統一驅動，跨 `CompactWidthThreshold`（新增的 styled property，預設 600 DIP）門檻才 Rebuild，band 內寬度變動不 thrash。寬度反應走繼承 `Bounds` direct property 的 `OnPropertyChanged`（class handler 對它不觸發）。`DetailEditMode` 保留為寬螢幕偏好（預設 `InCell`），桌面行為不變。零 head 接線——任何 head 在窄畫面渲染即自動 compact。
- **`ListView` 窄視窗卡片佈局（`Bee.UI.Avalonia`）** — 寬欄 grid 在手機下會裁切欄位、需橫向捲動。低於 `CompactWidthThreshold` 時改用 `ListBox` 卡片，每筆一張卡、可見欄位以「caption  value」逐行堆疊，直接讀 `DataRowView`（不走 reflection binding，iOS AOT 安全）。選取仍連動既有 toolbar 啟用流程。寬螢幕桌面 / 瀏覽器不受影響。
- **`RowEditPanel`（EditForm）響應式 + `RowEditDialog` 可調整大小（`Bee.UI.Avalonia`）** — 列編輯面板依宿主 top-level 寬度（螢幕寬度，非對話框自身寬度，避免桌面 `SizeToContent` 小視窗被誤判為 compact）1 ↔ 2 欄重排。桌面 native `RowEditDialog` 視窗改為 `CanResize=true`，開啟後把 `SizeToContent` 轉回 `Manual`，保留「初始依內容取大小」又能拖拉縮放。

### 修正

- **定義型別相容 AOT reflection-only XmlSerializer（`Bee.Definition`）** — AOT 目標（iOS，及任何 NativeAOT / MAUI-iOS 目標）無 `Reflection.Emit`，`XmlSerializer` 退回 reflection-only reader，載入任何 Bee XML 定義即崩。移除兩個不相容：(1) 集合有多個 public instance `Add`，reader 的 `Type.GetMethod("Add")` 丟 `AmbiguousMatchException` —— 每個定義集合現只剩單一 public instance `Add(T)`（基底介面 `Add(I…CollectionItem)` 改顯式介面實作；各集合便利 `Add(...)` 多載改為**同 namespace** 的擴充方法，呼叫端 `.Add(...)` 語法不變）；(2) owner-coupled 集合缺 `Activator.CreateInstance` 需要的 public 無參數建構子，為 12 個補 `public Xxx() : base()`。桌面 / 瀏覽器（WASM）走 code-gen 路徑本就正常、不受影響；XML 格式逐字不變（由 Definition + Base round-trip 測試驗證）。見 [ADR-025](../adr/adr-025-define-types-aot-xmlserializer-compat.md)。
- **`RowEditDialog` 在單視圖宿主走 overlay（`Bee.UI.Avalonia`）** — 列編輯原本只特判瀏覽器走 overlay、其餘一律開 native `Window`。iOS / Android single-view 不支援 native `Window`，編輯明細列會崩。改為能力導向：top-level 是 `Window`（桌面 classic-window lifetime）才開 native 模態，否則（瀏覽器 / iOS / Android single-view）一律走 `OverlayLayer`。移除會在行動端崩潰的 free-standing window fallback。
- **`FormView` 表單本體垂直捲動（`Bee.UI.Avalonia`）** — 內容原為無捲動垂直堆疊，單欄（compact）重排後變高即溢出視窗、下方控件無法觸及。改以 `DockPanel` 把錯誤訊息 + toolbar 釘頂端（Save / Cancel / Back 恆可達），表單本體放進垂直 `ScrollViewer`。明細 grid 行為不變。
- **`GridControl` lookup 可編輯 cell 放大鏡圖示（`Bee.UI.Avalonia`）** — 明細 grid 的 lookup 可編輯 cell（依 ADR-021 繞過 DataGrid 編輯管線、由顯示模板開窗）原只渲染純文字、無視覺提示，與主檔 `ButtonEdit` 不一致。可編輯分支現補上 muted 開窗放大鏡圖示；唯讀 / list 模式與 `ButtonEdit` 不變。
