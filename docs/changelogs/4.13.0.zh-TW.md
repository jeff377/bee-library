# Bee.NET 4.13.0 — 詳細發行說明

[English](4.13.0.md) ・ [繁體中文](4.13.0.zh-TW.md) ・ [← CHANGELOG](../../CHANGELOG.zh-TW.md)

本檔是 [CHANGELOG.zh-TW.md](../../CHANGELOG.zh-TW.md) 中 `[4.13.0]` 條目的詳細版。主 CHANGELOG 每項只列一行標題；本檔保留完整敘事——每項變更背後的 *why*、限制與設計脈絡。

> Bee.NET 仍處 pre-stable 演進階段。本版新增一套 ERP 級數值層，核心是單一想法：數值欄帶一個語意 **`NumberKind`**，由這個 kind ——而非欄位的儲存型別——決定值如何顯示、寫入時是否捨入、以及小數位數來自何處。其上再疊兩條 ERP 鐵則（**round-then-sum** 與 **preserve 永不捨入**）與兩個 runtime 解析的位數來源（**貨幣** 與 **計量單位**），鏡像 SAP 的 CURR/CUKY 與 QUAN/UNIT 家族。所有新增皆向後相容——新成員帶 `[DefaultValue]` 空、`CompanyInfo` 的新 MessagePack key 尾端追加、缺主檔則退框架預設——既有 schema、XML 與 wire payload 皆不受影響。完整設計理由與 SAP / Odoo 對照見 [ADR-026](../adr/adr-026-numeric-semantics-rounding.md)。

### 新增

- **欄位上的 `NumberKind` 語意（`Bee.Definition`）** — `FormField`（傳遞至 `LayoutFieldBase`）現帶一個 `NumberKind`：`Quantity`、`Weight`、`Amount`、`Percent`、`UnitPrice`、`Cost`、`ExchangeRate`。此 kind 是三件事的唯一真實來源——顯示格式字母（`N` / `P`）、捨入策略（`Round` vs `Preserve`）、位數來源（貨幣／單位／公司／系統固定）。框架預設：`Quantity` 0、`Weight` 3、`Amount` 2、`Percent` 2、`UnitPrice` / `Cost` 4、`ExchangeRate` 5。`NumberKindProfile` 暴露此 profile，取代孤立、0 caller 的 `NumberFormatPresets`。為何語意放在欄位上而非由 DB 型別推斷，見 [ADR-026](../adr/adr-026-numeric-semantics-rounding.md)。

- **捨入與格式化引擎（`Bee.Definition`）** — `NumberFormatResolver` 提供 `ResolveDecimals` / `ResolveFormat` / `RoundByKind` / `RoundCash`，`NumberFormatApplier.Bake` 把顯示格式蓋到 schema 上。兩條 ERP 鐵則在此把關：
  - **Round-then-sum。** `Round` 類的合計等於*已捨入明細之和*，絕不全精度加總後才一次捨。`RoundByKind` 對每筆明細（AwayFromZero）捨到其解析位數；呼叫端加總已捨入值，故 `Σ 明細 == 合計` 由構造保證。原幣與本幣各依自己的貨幣獨立捨入。
  - **兩層捨入。** 明細層捨到*自然*幣別／單位小數（`CurrencySettings` / `UnitSettings`）；選配的最終**現金捨入**（`RoundCash`，SAP T001R 式，如 CHF → 0.05）只作用於最終應付額，並把刻意產生的 `payable − total` 差額記入捨入科目。幣別小數系統層固定；只有現金捨入單位可公司覆寫。
  - **Preserve 永不回寫捨入值。** `UnitPrice` / `Cost` / `ExchangeRate` 經 `RoundByKind` 原值返回——位數僅供顯示，對來源值捨入會把誤差注入下游。
  - Bake 於 `SystemBusinessObject.LoadAndLocalizeSchema` 內的 **per-call clone** 上執行；process 共用快取絕不 mutate，作者手填的 `NumberFormat` 永遠優先。

- **多幣別：`CurrencySettings` + 逐欄 CUKY 綁定（`Bee.Definition`）** — 新增系統層幣別主檔（`DefineType.CurrencySettings`，curated ISO 4217，SAP TCURX 式），每個幣別一筆 `CurrencyItem`（`Code` / `Numeric` / `Rounding` / `Symbol` / `Name`），自然小數由其反推（JPY = 0、USD = 2、BHD = 3）。金額欄的位數於 **runtime 依其貨幣**解析、而非依公司：每個金額欄透過 `FormField.CurrencyField` 綁定貨幣鍵欄，單據主檔幣別放在 `FormSchema.CurrencyField`（慣例 `sys_currency`），解析優先序為 指定 `CurrencyField` → 主檔 `sys_currency` → 公司 `DefaultCurrency` → 框架 2。由於貨幣是會變動的單據資料，`Bake` 刻意*不* bake 金額格式——改把有效的貨幣參照欄蓋上去，讓 UI 逐列解析。主檔走既有 `GetDefine` 管道 ship 給 client；缺主檔則退 2 位。

- **計量單位：`UnitSettings` + 逐欄 UNIT 綁定（`Bee.Definition`）** — 與貨幣完全平行（SAP QUAN/UNIT 鏡像 CURR/CUKY）。系統層單位主檔（`DefineType.UnitSettings`，SAP T006 式）每個單位一筆 `UnitItem`（`Code` / `Decimals` / `Dimension` / `Name`），位數直存（KG = 3、PCS = 0）。`Quantity` / `Weight` 欄透過 `FormField.UnitField` 綁定單位欄（單位是 per 列——無主檔層單位概念）；解析優先序為 綁定 `UnitField` 值 → 公司位數 → 框架預設。Round-then-sum 依單位成立；混單位的欄無有意義的合計。

- **公司層數值模型（`Bee.Definition`）** — `CompanyInfo` 新增四個尾端追加成員：`NumberFormats`（`[Key(4)]`，`CompanyNumberFormats` 覆寫表，承載 `Percent` / `UnitPrice` / `Cost` 與數量/重量無單位時的 fallback）、`DefaultCurrency`（`[Key(5)]`，公司本幣）、`CashRounding`（`[Key(6)]`，`CompanyCashRounding` per-幣別現金捨入表）、`AllowedCurrencies`（`[Key(7)]`，per-公司幣別白名單，限縮單據貨幣下拉）。由四個新 `st_company` 欄位（`number_formats_xml`、`default_currency`、`cash_rounding_xml`、`allowed_currencies_xml`）承載，`CompanyRepository.GetById` 讀取。由於 key 尾端追加，既有 MessagePack payload 保持相容；欄空則退框架預設／空白名單（= 全幣別）。

- **`NumericEdit` 編輯器（`Bee.UI.Avalonia`）** — 新增 `ControlType.NumericEdit`（由 `LayoutColumnFactory` / `FieldEditorFactory` 對 `Short` / `Integer` / `Long` / `Decimal` / `Currency` 自動解析）。focus 顯完整儲存精度、blur 依欄位 `NumberFormat` 格式化、右對齊、無效輸入保留前值。關鍵是顯示捨入絕不回寫——`GetWriteBackValue` 解析後返回完整精度值。

- **貨幣／單位感知的 Grid 格式化（`Bee.UI.Avalonia`）** — `GridControl` **逐列**解析每個金額／數量格的格式，讀該列的 `CurrencyField` / `UnitField` 值（空退預設碼），故單一欄可讓 USD 與 JPY 列各以自己的位數呈現；設定留 null 則保留先前的欄級 baked 行為。`AmountColumnSummary.TryComputeTotal` 是 footer helper：對同幣別（或同單位）欄回合計、對混幣欄回 null——對齊 Odoo monetary-sum「混幣合計無意義」的行為。`NumericEdit` 與 Grid 共用同一套 per-cell 解析路徑。

### 附註

- **無破壞性變更。** 新的 `FormField` / `LayoutFieldBase` / `FormSchema` 成員預設空、不改既有 XML round-trip；`CompanyInfo` 新 key 尾端追加以維持 MessagePack wire 相容；新 `DefineType` 成員在無主檔時 fallback；`NumberFormatPresets`（0 production caller）重構為 `NumberKind` 模型，僅調整其自身測試。
- **`Bee.UI.Avalonia` 為試點。** `NumericEdit` 編輯器與 Grid 格式化先落地於 Avalonia 控件家族；MAUI / Blazor 移植另案跟進。
