# Bee.NET 4.12.1 — 詳細發行說明

[English](4.12.1.md) ・ [繁體中文](4.12.1.zh-TW.md) ・ [← CHANGELOG](../../CHANGELOG.zh-TW.md)

本檔是 [CHANGELOG.zh-TW.md](../../CHANGELOG.zh-TW.md) 中 `[4.12.1]` 條目的詳細版。主 CHANGELOG 每項只列一行標題；本檔保留完整敘事——每項變更背後的 *why*、限制與設計脈絡。

> Bee.NET 仍處 pre-stable 演進階段。4.12.0 讓 `Bee.Definition` 型別可在 AOT reflection-only XmlSerializer 下*反序列化*（無 `Reflection.Emit`）；本 patch 補上同一問題的另一半——*trimming*。在 trimmed 的 iOS / Android Release 建置下，linker 會砍掉「看似沒人用」的定義 metadata，因此本版隨套件內嵌一份 trimmer descriptor 把它保留下來。**無破壞性變更**——呼叫語法與磁碟上的 XML 格式皆不變。

### 修正

- **內嵌 trimmer descriptor，在 trim / AOT 下保留定義型別圖（`Bee.Definition`）** — Avalonia iOS / Android head 是 remote JSON-RPC client：`FormSchema`（及其他定義型別）以 XML 字串夾在 JSON-RPC payload 中傳到裝置，再由 `XmlCodec.Deserialize<T>` 在裝置端重建。在 trimmed 的 Release 建置下，linker 看不到參數無建構子、屬性 setter、強型別集合 `Add(T)` 的靜態呼叫端（唯一呼叫端是反射路徑，trimmer 追不到），於是把這些 metadata 砍掉。執行期表現為誤導性的 `XmlSerializeErrorDetails, 2, 2`（第 2 行第 2 欄正是根節點開頭——XML 沒壞，是型別 metadata 被裁掉）。

  本版新增 `src/Bee.Definition/ILLink.Descriptors.xml`，以邏輯資源名 `ILLink.Descriptors.xml`（.NET trimmer 會掃描的慣例名）內嵌，並以 `preserve="all"` wildcard root `Bee.Definition.*` 與 `Bee.Base.Collections.*`。由於它內嵌在發佈的組件中，trimmer 會**自動**套用至每一個下游 trimmed / AOT 消費端：外部 app 建置 trimmed 行動 head 時無需自行撰寫 linker 檔即可承襲此修正，也不需要在正常 transitive 相依 `Bee.Definition` 之外額外加 `PackageReference`。

  在 Android 模擬器 full-trim（`TrimMode=full`）建置上量測：無 descriptor 時 linker 砍掉 `Bee.Definition` 約 57%（212 KB → 91 KB），且序列化成員未被其他地方引用的定義型別可能因此遺失成員；有 descriptor 時組件被保留（約 208 KB），`FormSchema` round-trip 完整。不需實機即完成端到端驗證：Android 模擬器 full trim（round-trip 通過）、iOS device-target Release AOT build（編譯 0 錯誤，iOS linker 尊重 descriptor，linked `Bee.Definition.dll` 維持全量）、以及在 iOS 模擬器強制 reflection-only 序列化路徑（`RuntimeFeature.IsDynamicCodeSupported=false`，即 device AOT 鎖定的同一條路徑）下 round-trip 通過。

  採用廣域 wildcard root（而非逐型別 `[DynamicallyAccessedMembers]`）是刻意的：定義型別圖多型且子型別密集，逐型別註記容易漏——wildcard 一次涵蓋整個家族。相關：[ADR-025](../adr/adr-025-define-types-aot-xmlserializer-compat.md)（4.12.0 隨附的姊妹修正：reflection-only 反序列化相容）。完整調查與半 A／半 B 的脈絡見 [plan-mobile-release-trim-safe.md](../plans/plan-mobile-release-trim-safe.md)。
